2 Les bases

Le workflow typique de l'utilisation est

  1. ajout de fichier à l'index
  2. commit des modifications
  3. publication des modifications

2.1 Ajout de fichier à l'index

git add <nouveau fichier>

2.2 Confirmation des modifications

git commit

2.3 Publication des modifications

git push

3 Inspection du projet

4 Branches

  • Les branches servent à faire des développement sans toucher à la branche principale.

  • Permet de focuser son attention sur le dévellopement d'une fonctionalité.

  • Permet de mener des dévelloppements en parallèle.

Voir également les sections suivantes:

  • merge

4.1 Création d'une branche

4.1.1 Création rapide

git checkout -b <nom de la nouvelle branche>

4.1.2 Spécification de la branche d'ancrage

git checkout -b <fonctionnalité> <branche parent>

4.1.3 Synchronisation avec un serveur distant

git push --set-upstream origin <nom de la branche à mettre sur le serveur>

4.1.4 Inspection des branches distantes

git remote show origin

4.2 Destruction d'une branche

4.2.1 Destruction d'une branche locale

git branch -d <branche à élaguer>

4.2.2 Destruction d'une branche distante

git push origin :<branche distante>

4.2.3 Nettoyer localement les branches détruites sur le serveur d'attache

git remote prune origin

4.3 Astuces, bonnes pratiques et pièges à éviter

  • Réfléchir à vos choix de branche (utilité, objectif, lien avec les autres branches, ratio effort/avantage)
  • Attention à ne pas s'emmèler dans ses branches.
  • Une fonctionalité par branche
  • Utiliser les branches lorsque l'intégrité du master est critique (par exemple, pour un travail avec des collaborateurs).

6.3.1 Synchroniser un projet que l'on a forké

#Ajoutez à votre fork le remote du projet d'origine
$ git remote add projet_original_master git://github.com/repo/projet_original.git

#Mettez à jour votre fork en local
$ git fetch projet_original_master

#Fusionnez (merge) maintenant votre copie locale 
# avec le projet d'origine
$ git checkout master
$ git merge projet_original_master/master

#Validez vos changements
$ git commit -a -m "Synchronisation avec le projet original"

#Envoyez vos changements sur github
$ git push
